# Покращення V3 порівняно з V1

## Проблеми V1

### 1. Швидке перенавчання (overfitting) всіх моделей
Після розморожування backbone (епоха 6) train loss стрімко падав, а val loss зростав:
- ResNet-50: train_loss=0.11, val_loss=1.24 (розрив у ~10 разів)
- DenseNet-121: val_loss зріс вже з 7-ї епохи, early stop на 17-й
- EfficientNet-B0: val_loss зріс з 10-ї епохи
- ViT-B/16: val_loss зріс з 11-ї епохи

### 2. Колапс гібридної моделі
Hybrid CNN-Transformer повністю зколапсувався — передбачав тільки клас NV (мажоритарний, 67%) для всіх зображень:
- F1-score = 0.0 для 6 із 7 класів
- Val accuracy хаотично стрибала між ~1% і ~68%
- Loss ніколи не опустився нижче ~1.93 (близько до випадкового)

**Причина:** Transformer-частина (proj + cls_token + pos_embed + 4 шари TransformerEncoder, d_model=512) мала занадто багато випадково ініціалізованих параметрів. Градієнтний сигнал від класифікаційного loss не зміг подолати шум випадкової ініціалізації.

## Зміни V3

### 1. Зменшення learning rate (помірне)

| Параметр | V1 | V3 |
|---|---|---|
| LR_BACKBONE | 1e-4 | **5e-5** (×0.5) |
| LR_HEAD | 1e-3 | **7e-4** (×0.7) |

Менший LR зберігає pretrained ваги backbone краще, запобігаючи їх деструкції при fine-tuning.

### 2. Регуляризація

| Параметр | V1 | V3 |
|---|---|---|
| Label smoothing | 0 | **0.05** |
| Head dropout | 0 | **0.2** |
| Gradient clipping | немає | **max_norm=1.0** |

- **Label smoothing** запобігає надмірній впевненості моделі, що зменшує overfitting.
- **Dropout** перед класифікаційним шаром додає стохастичну регуляризацію.
- **Gradient clipping** стабілізує тренування, особливо для Transformer-моделей.

### 3. Перероблена гібридна архітектура

| Параметр | V1 | V3 |
|---|---|---|
| d_model | 512 | **256** |
| num_layers (Transformer) | 4 | **2** |
| Параметри моделі | 37.2M | **25.6M** |
| Ініціалізація cls_token, pos_embed | `torch.randn` | **`torch.zeros` + trunc_normal(std=0.02)** |
| LayerNorm після projection | немає | **є** |
| LayerNorm перед head | немає | **є** |
| Dropout перед head | немає | **0.2** |
| Ініціалізація proj, head | за замовчуванням | **Xavier uniform** |
| Freeze epochs | 5 | **10** |

**Ключові зміни:**
- Зменшення d_model та num_layers скоротило кількість випадково ініціалізованих параметрів.
- Правильна ініціалізація (trunc_normal з малим std замість randn) стабілізує початкові градієнти.
- LayerNorm після projection нормалізує вхід до Transformer, що критично для стабільності.
- Подовжене заморожування backbone (10 епох) дає Transformer-голові час навчитись осмислених attention-паттернів перед fine-tuning backbone.

### 4. Збільшення тренувальних параметрів

| Параметр | V1 | V3 |
|---|---|---|
| NUM_EPOCHS | 30 | **40** |
| EARLY_STOPPING_PATIENCE | 10 | **12** |

Більше епох компенсує повільнішу збіжність при зменшеному LR.

## Результати

### Загальне порівняння

| Модель | V1 Accuracy | V3 Accuracy | V1 F1 (macro) | V3 F1 (macro) |
|---|---|---|---|---|
| ResNet-50 | 79.0% | 76.7% | 0.651 | 0.623 |
| DenseNet-121 | 72.6% | 68.3% | 0.584 | 0.578 |
| EfficientNet-B0 | 70.9% | **79.3%** | 0.584 | **0.666** |
| ViT-B/16 | 70.6% | **78.5%** | 0.591 | **0.663** |
| Hybrid CNN-Transformer | 67.1% | **75.2%** | 0.115 | **0.637** |

### Динаміка тренування

**V1:** Val loss різко зростав після розморожування. Великий розрив train/val loss свідчив про запам'ятовування тренувальних даних.

**V3:** Val loss плавно знижувався і стабілізувався. Розрив train/val loss значно менший, що свідчить про кращу генералізацію.

### Висновок

ResNet-50 та DenseNet-121 показали трохи нижчі абсолютні числа, але їх V1 результати були завищені через overfitting (модель запам'ятовувала тренувальні дані, а не вчилась узагальнювати). V3 результати більш чесні та надійні.

Три моделі суттєво покращились, гібридна модель працює коректно, а ранжування моделей тепер відповідає очікуванням з літератури: EfficientNet > ViT > ResNet > Hybrid > DenseNet.
